  - Toggle visibility

- Button: "Remove"

- Removes key from storage

### State Management:

**Provider Data:**
```javascript
{
  id: string,
  name: string,
  type: enum, // openai, anthropic, google, mistral, ollama, custom
  apiKey: string, // stored securely
  baseUrl: string | null,
  status: enum, // valid, invalid, unverified, testing
  isDefault: boolean,
  lastUsed: Date | null,
  models: string[],
  customConfig: object | null
}
```

**Usage Data:**
```javascript
{
  providerId: string,
  date: Date,
  requestCount: number,
  inputTokens: number,
  outputTokens: number,
  cost: number
}
```

## ZFA Architecture Requirements

### Entity Definition

Uses **Provider** entity from UC-1.1.

**ProviderUsage Entity:**
```bash
zfa entity create -n ProviderUsage --field "id:String,providerId:String,date:DateTime,requestCount:int,inputTokens:int,outputTokens:int,cost:double" --json
```

### Clean Architecture Generation

```bash
zfa generate Provider --methods=get,getList,create,update,delete --data --vpc --state
zfa generate ProviderUsage --methods=get,getList,create --data --vpc --state
```

### Implementation Notes

1. **Secure Storage**: Override data source to use `flutter_secure_storage` for API keys
2. **Usage Tracking**: Create `ProviderUsage` record for each API call
3. **Cost Calculation**: Compute cost based on provider pricing model
